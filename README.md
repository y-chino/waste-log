# 廃棄物報告アプリ

マイポックス株式会社の各拠点・排出場所ごとの廃棄物重量をスマートフォン・PCからその場で記録し、Google スプレッドシートに集約するWebアプリケーションです。

Google Apps Script (GAS) をバックエンド、HTML/JavaScript をフロントエンドとして動作します。

---

## ファイル構成

```
/
├── code.gs           # バックエンド（GAS）
├── index.html        # フロントエンド（UI・ロジックすべて）
├── appsscript.json   # GAS マニフェスト（OAuthスコープ・デプロイ設定）
├── doc/              # 設計・仕様ドキュメント
└── bkup/             # バージョン別バックアップ
```

> **運用ルール**: GAS の制約上、スクリプトファイルは `code.gs` と `index.html` の2ファイル構成を維持してください。ファイルを分割するとデプロイ構成が変わります。

---

## スプレッドシートのシート構成

本アプリはバックエンドとして Google スプレッドシートを使用します。以下の6シートが必要です。

| シート名 | 役割 |
|---|---|
| `Master_Category` | 廃棄物種別マスタ（ID・名前・詳細・対象拠点） |
| `Master_Location` | 排出場所マスタ（ID・名前・所属・拠点） |
| `Master_User` | 組織ユーザー名簿キャッシュ（メール・氏名） |
| `UserSetting` | ユーザーごとのデフォルト場所設定 |
| `WastesData` | 報告データのログ（全16列） |
| `Config` | アプリ外部設定（URLなど） |

### Configシートのキー

| キー（A列） | 値（B列） |
|---|---|
| `FEEDBACK` | フィードバックフォームの URL |
| `SUMMARY` | 過去データ閲覧用スプレッドシートの URL |

## デプロイ手順

### 1. clasp でコードをプッシュ

```bash
clasp push
```

`code.gs` / `index.html` / `appsscript.json` の3ファイルが GAS プロジェクトに反映されます。

### 2. Webアプリとして公開

1. GAS エディタで「デプロイ」→「デプロイを管理」を開く
2. 既存デプロイを編集（バージョンを「新バージョン」に更新）して「デプロイ」
3. 初回は「新しいデプロイ」→ 種類:「ウェブアプリ」で以下を設定：
   - 「次のユーザーとして実行」→「アクセスしているユーザー」
   - 「アクセスできるユーザー」→「組織内の全員」

---

## 主な機能

- **拠点・場所別の廃棄物重量登録**（KP / HP / FP の3拠点対応）
- **カレンダーによる日付選択**（当月〜3ヶ月前の実績表示）
- **場所の切り替え**（スワイプ・ボタン・キーボード矢印キー）
- **登録履歴の確認**（当日・同一場所の報告ログ）
- **ユーザー設定の保存**（拠点・所属・場所を次回以降も維持）
- **初回アクセス時の自動ユーザー登録**（Google OAuth2 userinfo API で氏名取得・`Master_User` に自動追記）
