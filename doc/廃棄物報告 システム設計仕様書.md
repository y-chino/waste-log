# **廃棄物報告 システム設計仕様書**

作成日: 2026/03/22

バージョン: v1.0.7

## **1\. 概要**

本システムは、Mipox株式会社における拠点・排出場所ごとの廃棄物重量をスマートフォンから迅速に報告し、Google スプレッドシートに集約するためのWebアプリケーションである。Google Apps Script (GAS) をバックエンド、HTML/JavaScript (Tailwind CSS) をフロントエンドとする。

## **2\. システム構成**

| 項目 | 内容 |
| :---- | :---- |
| プラットフォーム | Google Apps Script (GAS) |
| データベース | Google スプレッドシート |
| フロントエンド | HTML5, Tailwind CSS, Font Awesome, JavaScript |
| 外部API | Admin SDK API (組織ユーザー情報の取得に使用) |

## **3\. UI/UX 仕様**

### **3.1 ヘッダーおよびナビゲーション**

* **共通ヘッダー** : 全画面でアイコン、タイトル、プロフィール位置を完全に固定し、画面遷移時の視覚的負荷を最小化する。  
* **アプリタイトル優先表示** : アプリタイトルは常に全表示し、画面幅が狭い場合は所属名側を省略（Truncate）する。  
* **Sticky（追従）表示** :  
  * 報告画面（Main View）：ヘッダーおよび日付選択エリアを上部に固定する。  
  * 設定画面（Registration View）：ヘッダーのみを上部に固定する。  
* **オートルーティング** :  
  * 起動時、場所情報（Step 03）まで登録済みのユーザーは自動的に「報告画面」を表示。  
  * 未登録、または設定が不完全なユーザーは自動的に「初期設定画面」を表示。

### **3.2 初期設定フロー（段階的入力制御）**

* **Step 01 (拠点)** : 起動時は未選択（保存済みの場合はその値）。選択されるまで Step 02 以降は操作不能。  
* **Step 02 (所属)** : 拠点選択後に有効化してフォーカスが当たる。所属が選択されるまで Step 03 は操作不能。  
* **Step 03 (場所)** : 所属選択後に有効化してフォーカスが当たる。  
* **送信制限** : Step 03 まで全ての項目が確定するまで保存ボタンを無効化（Disabled）し、不完全な設定を防止する。  
* **保存時演出** : 保存処理中はボタンラベルを「保存中...」に変更し、処理状況を可視化する。

### **3.3 カレンダー UI**

* **高密度設計** : セル間隔を最小化し、限られた画面領域で1ヶ月の俯瞰性を確保する。  
* **報告状況の可視化** : 報告データが存在する日付は、薄緑（Emerald 50）の背景を適用し、枠線を排除したフラットデザインとする。  
* **数量表記** : 報告済みの日の下部には、その日の合計重量（Kg）を太字で表示する。  
* **週表示** : 当月の日付が含まれる「日曜日」から「土曜日」までの範囲をグリッド表示する。翌月の不要な空行は表示せず、常に週単位で完結させる。
* **月移動ボタンの配置** : カレンダーヘッダーの月移動ボタン（前月・次月）は flex レイアウトにより左右端に配置し、下部の場所ナビバーのボタンと縦位置を揃える。

### **3.4 PC / デスクトップ対応**

* **場所ナビボタン** : 同一所属内の場所切り替えを、スワイプに加えて左右の `‹` / `›` ボタンでも操作可能。複数場所が存在する場合のみ表示し、端（先頭・末尾）では非活性（Disabled）になる。
* **キーボード操作** : メインビュー表示中かつ入力フォーカスがない状態で、`←` `→` キーによる場所切り替えが可能。

## **4\. データベース構成 (Google Sheets)**

### **4.1 シート構成**

| シート名 | 役割 | 主要項目 |
| :---- | :---- | :---- |
| Master\_Category | 廃棄物種別マスタ | ID, 名前, 詳細, 対象拠点 |
| Master\_Location | 排出場所マスタ | ID, 名前, 部門(Dept), 拠点(Base) |
| Master\_User | ユーザー名簿キャッシュ | メールアドレス, 名前 |
| UserSetting | ユーザー設定 | メール、拠点、場所ID |
| WastesData | 報告ログ | ID, カテゴリID, 重量, 日付, 部門, 更新者(Email), 更新者名, 更新日時等 |
| Config | アプリ外部設定 | キー (Key), 値 (Value), 備考 |

### **4.2 WastesDataシートの詳細構造**

集約・検索効率化のための自動計算列を含む15列以上で構成する。

| 列番号 | 項目名 | 内容 |
| :---- | :---- | :---- |
| 1 | ID | ユニークID（行番号+α） |
| 2 | カテゴリID | Master\_CategoryのID |
| 3 | カテゴリ名 | 廃棄物種別名称 |
| 4 | 計測値 | 重量 (Kg) |
| 5 | 日付 | 報告対象日 (yyyy-mm-dd) |
| 6 | 拠点名 | KP / HP / FP |
| 7 | 排出元ID | Master\_LocationのID |
| 8 | 排出元名 | 場所名 (例: CR1) |
| 9 | 排出元所属 | 部門名 (例: B01) |
| 10 | 年 | 自動計算: yyyy年 |
| 11 | 月 | 自動計算: yyyy年mm月 |
| 12 | 日付Number | 自動計算: yyyymmdd (数値型) |
| 13 | 更新者 | Session UserのEmail |
| 14 | 更新者名 | Master\_Userから取得した氏名 |
| 15 | 更新日時 | 書き込み時のタイムスタンプ |
| 16 | タイトル | 自動生成: 拠点\_場所\_カテゴリ\_日付 |

## **5\. 主要ロジック (Backend/Core)**

### **5.1 起動処理 (Hydration)**

* **データハイドレーション** : doGet は `initialPayload = "null"` として HTML を返す。クライアント側の `initApp` が起動後に非同期で `getInitialDataForApp` を呼び出すことで、ページの骨格 UI を先行表示し体感速度を向上させる。  
* **実績データの取得制限 (3暦月制限)** : 起動時は「当月・先月・先々月」の3暦月分のデータのみを読み込む。  
* **遅延読み込み (Lazy Loading)** : カレンダーで3ヶ月より前の月が表示された際、サーバーにリクエストを送り追加取得する。取得済みデータはメモリにキャッシュする。  
* **データクレンジング** : マスタ取得時に文字列に対して .trim() を実行。メールアドレスは小文字（.toLowerCase()）に統一し照合エラーを防止する。

### **5.2 ユーザー特定と名前の記録**

* **自動名前取得** : メールアドレスをキーに、Master\_User シートから名前を検索する。  
* **組織同期** : Master\_User に存在しない場合、Admin SDK API を介して組織内からフルネームを自動取得し、シートに追記する。  
* **手動同期** : 管理者は syncOrganizationUsers 関数を実行することで、組織名簿を一括更新可能。

### **5.3 データ保存・更新・フィルタリング**

* **拠点によるカテゴリフィルタリング** : targetBase が現在の拠点と一致（または空欄で共通）する場合のみ表示。保存時の最終ガードにも適用。  
* **足し込み更新（上書き更新）** : 同一日、同一場所、同一カテゴリの報告は、既存レコードの重量を最新の合計値で更新する。  
* **フィードバック機能** : 「アプリ説明」内の「フィードバックを送る」ボタンは、Configシートの `FEEDBACK` キーの値を URL として参照する。同様に「過去データを見る」は `SUMMARY` キーを参照する。

## **6\. 設定ページの重要ロジック**

### **6.1 ロック・ロック解除メカニズム**

* **ロック状態（LOCKED）** : 設定保存後は読み取り専用となり、バッジに「LOCKED」と表示。  
* **解除フロー（UNLOCKED）** : 「ロックを解除する」ボタンで編集可能へ遷移。バッジは「UNLOCKED」となりストライプ背景を適用。

### **6.2 フォーカス制御フロー**

拠点選択 → 所属にフォーカス → 場所選択 → 保存ボタン活性化。

## **7\. 日付セクションとカレンダー制御**

### **7.1 カレンダーバーの機能構成**

* **今日ボタンエリア** : 現在日付に即座にフォーカスし、月と選択状態をリセット。  
* **開閉操作グループ** : 中央ラベルから右端アイコンまでの全領域で開閉を制御。

### **7.2 数値表記および自動開閉**

* **未報告（Pending）数値** : 未送信データがある場合、オレンジ色で \+数値 を表示。  
* **実績（Recorded）数値** : 保存済みの合計重量を緑色で表示。  
* **自動開閉** : 「空の日付」選択時は自動で閉じ、データがある日付選択時は展開状態を維持。

## **8\. 計量アクションとカード表記ルール**

### **8.1 計量パネル（Slider Panel）**

* **入力同期** : 数値、ボタン、スライダーが完全同期。スライダー範囲 0 〜 100 Kg。  
* **排他的編集モード** : 「既登録分を修正」のON/OFFにより、実績値の修正か今回分の追加入力を切り替える。  
* **フィードバック** : 確定時、カレンダーが閉じている場合は自動展開し、数値を即座に反映（Pending色）。

### **8.2 廃棄物カードおよび履歴表示**

* **表記ルール** : 追加計量時は合計重量の真上にオレンジ色で \+数値 と表記。  
* **登録履歴表示** : 当日・同一場所の実績をカードエリア下部にアコーディオン形式で表示。時刻、カテゴリ、数値、ユーザーを表示。

## **9\. パフォーマンス最適化要件**

1. ヘッダー・日付バー等の骨格UIを最優先。  
2. 廃棄物カテゴリカード、カレンダーの順に描画。  
3. 履歴リストを非同期で描画（10ms遅延のLazy Rendering）。  
4. 一括送信時はプログレスバーで進捗（n/m）を表示。

## **10\. セキュリティ・制限**

* **管理画面制限** : 画面幅 768px 未満（スマホ）からの管理遷移をブロック。  
* **管理者導線** : アプリアイコンの長押し（600ms）による隠し遷移。
