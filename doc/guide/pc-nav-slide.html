課題

スマホでしか場所を切替できなかった

スマホ ✓ 対応済

カードエリアを左右スワイプすることで、同一所属内の場所をスムーズに切り替えられた。

PC ✗ 未対応

タッチ操作がないため、場所を切り替える手段が存在しなかった。
最初に設定した場所しか使えない状態。

解決策

2つの操作方法を追加

‹ › ボタン

場所名バーの左右に矢印ボタンを配置。クリックで前後の場所に切り替え。

場所：Mirai
キーボード操作

メインビュー表示中、入力フォームに フォーカスがない状態でキー操作が有効。

前の場所 / 次の場所
ボタン詳細仕様

‹ › ボタンの表示ルール

1

複数場所がある場合のみ表示

同一所属（Dept）内に場所が1つだけの場合、ボタンは非表示（invisible）。レイアウトは変化しない。

2

端では非活性（Disabled）

先頭では ‹ ボタンが、末尾では › ボタンが 30% 透明になりクリック不可。

3

カレンダーボタンと縦位置を揃えた

月移動ボタンも同時に flex 化し、場所ナビのボタンと垂直に揃えて視覚的統一感を確保。

キーボード操作詳細

有効・無効の条件

有効になる条件（AND）

メインビューが表示されている
計量スライダーが開いていない
INPUT / SELECT 等にフォーカスがない

無効になるケース

計量パネル（スライダー）表示中
設定画面・説明画面を表示中
数値入力フォームを操作中
