# **廃棄物報告アプリ デモデータ定義ガイド**

本ドキュメントは、アプリの「デモモード」で使用される getDemoData 内のレコード構成と、今後の開発でデータを追加する際の指針をまとめたものです。

## **1\. データの基本構造**

デモデータは以下の3つの拠点（Base）を軸に構成されています。各拠点の「現場感」を出すために、固有の部門名（Dept）と場所名（Location）を割り当てています。

| 拠点名 (Base) | 特性・コンセプト | 主な廃棄物カテゴリ |
| :---- | :---- | :---- |
| **HP (Main Plant)** | 本社・技術センター。クリーンルームや研究開発、精密工程のイメージ。 | 有機溶剤、ガラス、再生フィルム |
| **KP (Kofu Plant)** | 工場。大型プレス機、組立ライン、物流センターのイメージ。 | 可燃ゴミ、木くず、廃プラ |
| **FP (Fuji Plant)** | 製造・加工拠点。化学工程、研磨、多種多様な製造ラインのイメージ。 | 廃液、固形燃料、研磨材 |

## **2\. 場所マスタ (Locations)**

各拠点につき3つ以上の所属（Dept）を設け、それぞれの所属に5つの場所を配置しています。

### **HP: 技術・精密拠点**

* **B01**: CR1  
* **B04**: Mirai, MigakuとHakaru

### **KP: 生産・物流拠点**

* **11B**: 物流課, 事務所, プレス1号機

### **FP: 化学・加工拠点**

* **製造一課**: 調合  
* **製造三課**: 電着C  
* **製造四課**: スリッター3機

> **Note**: 現行の `getDemoData()` は動作確認用の最小構成。§4 の開発指針（各部門5場所以上）に沿った拡充は今後の課題。

## **3\. カテゴリマスタ (Categories)**

targetBase プロパティを利用して、拠点ごとに最適な選択肢が表示されるようフィルタリングしています。

| ID | カテゴリ名 | 対象拠点 | 詳細・備考 |
| :---- | :---- | :---- | :---- |
| 1-12 | 普通ごみ, 木くず, 有機溶剤等 | HP | 技術的な廃棄物を含む |
| 13-17 | 廃液, 固形燃料, 一斗缶等 | FP | 化学・液体廃棄物中心 |
| 18-21 | 可燃ごみ, 木くず, 廃プラ等 | KP | 工場・物流由来の廃棄物中心 |

## **4\. 開発時のデータ追加指針**

今後、開発やテストでレコードを追加する際は、以下の「センス」を維持してください。

1. **リアリティのある命名**:  
   * 単なる「場所A」ではなく、「〇〇ライン」「〇〇バース」「待機室」など、工場の現場で使われていそうな名称を採用する。  
2. **拠点ごとのカラー分け**:  
   * 拠点が切り替わった際に、並んでいる場所名やカテゴリ名を見るだけで「今どの拠点にいるか」を直感的に判断できるようにする。  
3. **スワイプテストの考慮**:  
   * 各部門（Dept）に5つ以上の場所を置くことで、UIのドットインジケーターやスワイプ切り替えの挙動（最後のページまでの遷移など）を常に確認可能にする。

これらのデータは index.html 内の getDemoData() 関数にて管理されています。
