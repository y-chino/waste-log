# 廃棄物報告システム：氏名取得・変換ロジック詳細解説書

本ドキュメントでは、Google Apps Script + Google スプレッドシートによる本システムにおいて、ユーザーの「メールアドレス」から「氏名（表示名）」を特定・変換し、報告データ（WastesData シート）に書き込むまでの詳細なメカニズムを解説します。

---

## 1. 全体像（データフロー）

本システムにおける氏名取得は、**Google アカウントのアクティブセッション**を起点とし、**Google OAuth2 userinfo API** を介してスプレッドシート上に構築された **「組織ユーザー名簿キャッシュ（Master_User シート）」** と照合する仕組みを採用しています。

```
[1. 起動 / 報告処理の実行]
  └─ Session.getActiveUser().getEmail() により、Googleアカウントの「メールアドレス」を取得
       │
[2. キャッシュマスタ照合] getOrRegisterUserName_()
  ├─ Master_User シート内の「メールアドレス（A列）」を走査
  │    ├─ 【一致あり】 ──> 「名前（B列）」を取得して決定
  │    └─ 【一致なし】 ──> [3. userinfo API で自己取得] へ
  │
[3. userinfo API による名前取得（初回アクセス時のみ）]
  ├─ ScriptApp.getOAuthToken() でアクセストークン取得
  ├─ GET https://www.googleapis.com/oauth2/v3/userinfo を呼び出し
  ├─ LockService で同時書き込み競合を防止
  ├─ 【取得成功】 ──> Master_User シートに追記 ──> [4. 処理続行]
  └─ 【取得失敗】 ──> [4. フォールバック] へ
       │
[4. フォールバック（最終防衛策）]
  └─ 名簿登録なし / API エラーの場合
       └─ メールアドレスの @ より前の部分（ローカルパート）を暫定氏名とする
       │
[5. データ書き込み] executeSaveReport()
  └─ 最終特定された氏名（または暫定名）を WastesData シートの「更新者名」列に記録
```

---

## 2. 詳細ロジックとプログラム内の実装

### ① 起点：アクティブユーザー（メールアドレス）の取得

ユーザーが Web アプリケーション（index.html）を開いた際、GAS サーバーサイドで現在のユーザーのセッション情報を取得します。

```javascript
// code.gs 内の取得部分
var userEmail = Session.getActiveUser().getEmail() || "anonymous@mipox.co.jp";
```

- **仕組み**: ウェブアプリは `executeAs: USER_ACCESSING`（アクセスしているユーザーとして実行）で動作するため、`getActiveUser()` はアクセスしたユーザー自身のメールアドレスを返します。

### ② 変換の核心：名簿キャッシュとの照合（getOrRegisterUserName_）

取得したメールアドレスをキーに、スプレッドシート上の `Master_User`（ユーザー名簿キャッシュ）シートと突き合わせます。

```javascript
function getOrRegisterUserName_(email) {
  if (!email) return null;
  var lowEmail = email.toLowerCase();
  var ss = SpreadsheetApp.getActiveSpreadsheet();
  var sheet = ss.getSheetByName(SHEETS.USER_MASTER);

  // キャッシュシートを検索
  if (sheet) {
    var data = sheet.getDataRange().getValues();
    for (var i = 1; i < data.length; i++) {
      // メールアドレスを小文字に統一して完全一致で比較
      if (String(data[i][0]).toLowerCase() === lowEmail) {
        return data[i][1]; // 一致した場合、B列の「名前」を返す
      }
    }
  }
  // 一致なし → [3. userinfo API] へ
}
```

- **工夫点**: 照合エラーを防ぐため、メールアドレスはすべて小文字（`.toLowerCase()`）に変換してから判定しています。

### ③ userinfo API による名前取得（初回アクセス時のみ）

`Master_User` にユーザーが見つからない場合（初回アクセス時）、Google OAuth2 の userinfo エンドポイントを呼び出して、アクセス中のユーザー自身の表示名を取得します。Admin SDK（管理者権限）は不要です。

```javascript
try {
  var token = ScriptApp.getOAuthToken();
  var res = UrlFetchApp.fetch(
    'https://www.googleapis.com/oauth2/v3/userinfo',
    { headers: { 'Authorization': 'Bearer ' + token }, muteHttpExceptions: true }
  );
  var person = JSON.parse(res.getContentText());
  var name = person.name || null;

  if (name) {
    // LockService で同時書き込み競合を防止
    var lock = LockService.getScriptLock();
    lock.waitLock(5000);
    try {
      if (!sheet) {
        sheet = ss.insertSheet(SHEETS.USER_MASTER);
        sheet.appendRow(['メールアドレス', '名前']);
        sheet.getRange('1:1').setFontWeight('bold').setBackground('#f3f3f3');
        sheet.setFrozenRows(1);
      }
      // 書き込み直前に再確認（二重登録防止）
      var latest = sheet.getDataRange().getValues();
      var alreadyAdded = latest.some(function(r) {
        return String(r[0]).toLowerCase() === lowEmail;
      });
      if (!alreadyAdded) sheet.appendRow([email, name]);
    } finally {
      lock.releaseLock();
    }
    return name;
  }
} catch(e) { }
```

- **管理者権限不要**: `oauth2/v3/userinfo` は `userinfo.profile` スコープで動作するため、Google Workspace の管理者でなくても使用できます。
- **初回のみ**: 2回目以降は `Master_User` のキャッシュが使われるため API 呼び出しは発生しません。
- **重複防止**: `LockService.getScriptLock()` で同時アクセスによる二重追記を防いでいます。

### ④ フォールバック（名前が解決できなかった場合）

何らかの理由（API エラー等）で名前が取得できなかった場合、メールアドレスのローカルパートを暫定名として使用します。

```javascript
// 例：y-chino@mipox.co.jp ──> "y-chino" に変換
var userName = getOrRegisterUserName_(userEmail);
if (!userName) { userName = userEmail.split('@')[0]; }
```

### ⑤ データベース（WastesData）への書き込み

最終的に決定された氏名（`userDisplayName`）は、`executeSaveReport` 内で報告レコードの要素としてシートに書き込まれます。

```javascript
var userEmail = Session.getActiveUser().getEmail() || "anonymous@mipox.co.jp";
var userName = getOrRegisterUserName_(userEmail);
var userDisplayName = userName || (userEmail ? userEmail.split('@')[0] : "不明ユーザー");

// スプレッドシートの行追加時、更新者名として書き込む
sheet.appendRow([
  ...,
  userDisplayName, // 更新者氏名（または暫定ID）
  formattedNow,    // 書き込み日時タイムスタンプ
  ...
]);
```

## 3. 必要な OAuth スコープ（appsscript.json）

```json
"oauthScopes": [
  "https://www.googleapis.com/auth/spreadsheets",
  "https://www.googleapis.com/auth/script.external_request",
  "https://www.googleapis.com/auth/userinfo.profile",
  "https://www.googleapis.com/auth/userinfo.email",
  "https://www.googleapis.com/auth/script.scriptapp"
]
```

スコープを変更した場合、既存ユーザーは `myaccount.google.com/permissions` でアプリのアクセス権を削除し、再アクセスして再認証する必要があります。
