# **廃棄物報告システム：氏名取得・照合および変換ロジック詳細解説書**

本ドキュメントでは、本システム（Google Apps Script \+ Google スプレッドシート）におけるユーザーの「メールアドレス」から「氏名（表示名）」を特定・変換し、報告データ（WastesData シート）に書き込むまでの詳細なメカニズムを解説します。

## **1\. 全体像（データフロー）**

本システムにおける氏名取得は、「Google Workspaceのアクティブセッション」**を起点とし、**「Admin SDK API」**を介してスプレッドシート上に構築された**「組織ユーザー名簿キャッシュ（Master\_User シート）」と照合する仕組みを採用しています。

以下に大まかなデータフローを示します。

\[1. 起動 / 報告処理の実行\]  
  └─ Session.getActiveUser().getEmail() により、Googleアカウントの「メールアドレス」を取得  
       │  
\[2. キャッシュマスタ照合\] getOrRegisterUserName\_()  
  ├─ Master\_Userシート内の「メールアドレス（A列）」を走査  
  │    ├─ 【一致あり】 ──\> 「名前（B列）」を取得して決定  
  │    └─ 【一致なし】 ──\> \[3. 組織同期処理\] へ  
  │  
\[3. 組織同期処理（Master\_Userシート不在 or 初回照合時）\] syncOrganizationUsers()  
  ├─ Google Workspaceの「Admin SDK (Directory API)」を呼び出し  
  ├─ 組織内（ドメイン内）の全ユーザーの \[メールアドレス\] と \[FullName\] を一括取得  
  └─ Master\_Userシートを最新情報で上書き更新 ──\> \[2. 照合\] を再実行  
       │  
\[4. フォールバック（最終防衛策）\]  
  └─ 名簿に登録がない、またはAPIエラー等の場合  
       └─ メールアドレスの @ より前の部分（ローカルパート）を暫定氏名とする  
       │  
\[5. データ書き込み\] executeSaveReport()  
  └─ 最終特定された氏名（または暫定名）を WastesData シートの「更新者名」列に記録

## **2\. 詳細ロジックとプログラム内の実装**

各プロセスにおける具体的なプログラムの挙動と役割は以下の通りです。

### **① 起点：アクティブユーザー（メールアドレス）の取得**

ユーザーがWebアプリケーション（index.html）を開いた、あるいは「報告する」ボタンを押した際、GASのサーバーサイドで現在実行しているユーザーのセッション情報を取得します。

// code.gs 内の取得部分  
var userEmail \= Session.getActiveUser().getEmail() || "anonymous@mipox.co.jp";

* **仕組み**: ユーザーがGoogleアカウントにログインした状態でGASアプリにアクセスしているため、Googleのセッションから直接、安全にメールアドレスが取得されます。

### **② 変換の核心：名簿キャッシュとの照合（getOrRegisterUserName\_）**

取得したメールアドレスをキーに、スプレッドシート上の Master\_User（ユーザー名簿キャッシュ）シートと突き合わせます。

function getOrRegisterUserName\_(email) {  
  if (\!email) return null;  
  var lowEmail \= email.toLowerCase();  
  var ss \= SpreadsheetApp.getActiveSpreadsheet();  
  var sheet \= ss.getSheetByName(SHEETS.USER\_MASTER);  
    
  // マスタシート自体が存在しない場合は、即座に組織同期を実行してシートを作る  
  if (\!sheet) {   
    syncOrganizationUsers();   
    sheet \= ss.getSheetByName(SHEETS.USER\_MASTER);   
  }  
    
  var data \= sheet.getDataRange().getValues();  
  for (var i \= 1; i \< data.length; i++) {  
    // メールアドレスを小文字に統一（toLowerCase()）して完全一致で比較  
    if (String(data\[i\]\[0\]).toLowerCase() \=== lowEmail) {   
      return data\[i\]\[1\]; // 一致した場合、2列目の「名前」を返す  
    }  
  }  
  return null; // 一致するユーザーが名簿に存在しない場合  
}

* **工夫点**: 照合エラーを防ぐため、メールアドレスはすべて小文字（.toLowerCase()）かつ文字列に変換してから判定されています。

### **③ バックグラウンド同期：Google Workspace組織情報との連携（syncOrganizationUsers）**

本システムの強みは、手動のアドレス帳登録を必要とせず、Google Workspaceのドメイン情報から名簿を自動生成・更新できる点にあります。これには **Admin SDK API (Directory API)** が使われています。

function syncOrganizationUsers() {  
  ...  
  // Admin SDK が有効化されているかチェック  
  if (typeof AdminDirectory \=== 'undefined') { return "エラー: Admin SDK API が有効化されていません。"; }  
  ...  
  var users \= \[\];  
  var pageToken;  
  try {  
    do {  
      // 組織のユーザーリストを最大500件ずつページネーションを回して取得  
      var response \= AdminDirectory.Users.list({  
        customer: 'my\_customer',  
        maxResults: 500,  
        pageToken: pageToken,  
        orderBy: 'email',  
        viewType: 'domain\_public'   
      });  
      if (response.users && response.users.length \> 0\) {  
        response.users.forEach(function(user) {  
          // \[プライマリメール(小文字化), フルネーム\] のペアを配列に格納  
          users.push(\[user.primaryEmail.toLowerCase(), user.name.fullName\]);  
        });  
      }  
      pageToken \= response.nextPageToken;  
    } while (pageToken); // 次のページがある限りループ

    if (users.length \> 0\) {  
      // Master\_User シートの2行目以降（古いデータ）をクリアして最新情報に一括置換  
      var lastRow \= sheet.getLastRow();  
      if (lastRow \> 1\) { sheet.getRange(2, 1, lastRow \- 1, 2).clearContent(); }  
      sheet.getRange(2, 1, users.length, 2).setValues(users);  
      SpreadsheetApp.flush();   
      return "同期完了: " \+ users.length \+ " 名を登録しました。";  
    }  
    ...  
}

* **仕組み**: 同一Google Workspaceドメイン（Mipox社内）に属するすべてのユーザーの登録名（user.name.fullName）とアドレス（user.primaryEmail）をAPIで引っこ抜き、スプレッドシートの Master\_User シートを更新（キャッシュ）します。

### **④ フォールバック（名前が解決できなかった場合）**

何らかの理由（新入社員で組織同期が未完了、外部ドメインのユーザー等）により Master\_User シートから名前が取得できなかった場合、そのままメールアドレスを表示するのではなく、ローカルパートを切り出す処理が挟まれています。

// 例：y-chino@mipox.co.jp ──\> "y-chino" に変換  
var userName \= getOrRegisterUserName\_(userEmail);  
if (\!userName && userEmail) {   
  userName \= userEmail.split('@')\[0\];   
}

これにより、画面上やデータベース上に極力無機質なメールアドレスが直接露出するのを防いでいます。

### **⑤ データベース（WastesData）への書き込み**

最終的に決定された氏名（userDisplayName）は、データ保存用の executeSaveReport 内で報告レコードの1要素としてシートに書き込まれます。

// 送信処理（executeSaveReport）の一部  
var userEmail \= Session.getActiveUser().getEmail() || "anonymous@mipox.co.jp";  
var userName \= getOrRegisterUserName\_(userEmail);  
var userDisplayName \= userName || (userEmail ? userEmail.split('@')\[0\] : "不明ユーザー");

...  
// スプレッドシートの行追加時、更新者名として「userDisplayName」を書き込む  
sheet.appendRow(\[  
  ...,  
  userDisplayName, // 更新者氏名（または代替ID）  
  formattedNow,    // 書き込み日時タイムスタンプ  
  ...  
\]);

## **3\. 仕様書と実コードの差異（注意点）**

「システム設計仕様書」の項目「**4.2 WastesDataシートの詳細構造**」と、実際の code.gs（プログラム）の書き込み処理を比較すると、書き込む列の位置に若干の差異（仕様書のアップデート漏れ、またはコードの最適化によるもの）が見られます。

* **仕様書の記述**:  
  * 13列目: 更新者（Email）  
  * 14列目: 更新者名（Master\_Userから取得した氏名）  
  * 15列目: 更新日時  
* **実際のプログラム挙動 (code.gs)**:  
  * executeSaveReport() では、変換後の氏名（userDisplayName）を直接 appendRow の **13列目** に代入し、**14列目** に更新日時（formattedNow）を代入しています（プログラム側ではアクティブユーザーのメールアドレス自体は WastesData に書き込まず、特定された氏名のみを書き込んでいます）。
